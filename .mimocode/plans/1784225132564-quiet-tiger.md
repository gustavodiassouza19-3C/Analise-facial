# FaceMax - Consolidação Final para Produção

## Visão Geral

O projeto tem **dois sistemas de dados paralelos** que precisam ser unificados para a produção:
- **Supabase** (frontend): `profiles`, `analyses`, `biometric_data` — usa Auth + RLS
- **Backend SQLAlchemy** (Python): `users`, `facial_analyses`, `analysis_categories` — usa python-jose JWT

A estratégia é: **Supabase é a fonte de verdade para auth, profiles, analyses e storage.** O backend é usado apenas para: detecção de face (MediaPipe), cálculos geométricos, e chamada à IA (OpenRouter). O profissional fará o veredito diretamente no Supabase (via frontend), não no backend.

## Área 1: Autenticação e Redirecionamento por Role

### 1.1 `frontend/src/pages/LoginPage.jsx` — Redirecionar por role após login

**Problema**: Sempre redireciona para `/dashboard` independentemente do role.

**Mudança**: Após `login()` retornar `success`, chamar `fetchProfile()` e ler `user.role` antes de navegar.

```jsx
// Após login bem-sucedido, ler o profile para decidir rota
if (result.success) {
  // O AuthContext já busca o profile via onAuthStateChange.
  // Mas pode haver race condition — aguardar profile com um pequeno polling.
  const selectedPlan = localStorage.getItem('selected_plan');
  if (selectedPlan) {
    navigate('/checkout-simulation');
  } else {
    // Ler o user do contexto após um tick para garantir que o profile já foi fetchado
    await new Promise(r => setTimeout(r, 100));
    const currentUser = useAuth().user; // ou passar via callback
    if (currentUser?.role === 'professional' || currentUser?.role === 'admin') {
      navigate('/professional/dashboard');
    } else {
      navigate('/dashboard');
    }
  }
}
```

**Abordagem mais limpa**: Modificar `AuthContext` para expor uma função `getRole()` que espera o profile ser carregado, OU usar `useEffect` que reage a `user.role` mudando de undefined para um valor.

**Decisão**: Usar `useEffect` no LoginPage que observa `user` do contexto:
```jsx
const { user } = useAuth();
useEffect(() => {
  if (!user || !user.role) return;
  const selectedPlan = localStorage.getItem('selected_plan');
  if (selectedPlan) {
    navigate('/checkout-simulation');
  } else if (user.role === 'professional' || user.role === 'admin') {
    navigate('/professional/dashboard');
  } else {
    navigate('/dashboard');
  }
}, [user, navigate]);
```

### 3.3 `frontend/src/lib/supabase/storage.js` — Fix upload parcial

**Problema**: Se slot 1 (front) faz upload mas slot 2 (left) falha, slot 1 fica órfão.

**Mudança**: Adicionar cleanup no loop:
```js
export async function uploadAnalysisPhotos(userId, analysisId, photos) {
  const slots = ['front', 'left', 'right'];
  const urls = {};
  const uploadedPaths = [];

  try {
    for (const slot of slots) {
      const file = photos[slot];
      if (!file) continue;
      const blob = await fetch(file).then((r) => r.blob());
      const ext = blob.type.includes('png') ? 'png' : 'jpg';
      const fileObj = new File([blob], `${slot}.${ext}`, { type: blob.type });
      const url = await uploadAnalysisPhoto(userId, analysisId, slot, fileObj);
      urls[`photo_${slot}_url`] = url;
      uploadedPaths.push(`${userId}/${analysisId}_${slot}.${ext}`);
    }
    return urls;
  } catch (err) {
    // Cleanup: remover todos os arquivos que já foram uploaded
    if (uploadedPaths.length > 0) {
      const supabase = createClient();
      await Promise.allSettled(
        uploadedPaths.map(path => supabase.storage.from('analysis-photos').remove([path]))
      );
    }
    throw err;
  }
}
```

### 3.4 `frontend/src/components/evaluation/FaceAnalyzer.jsx` — Validação no frontend

Adicionar validação de magic bytes antes do upload:
```js
const validateImageMagicBytes = async (dataUrl) => {
  const response = await fetch(dataUrl);
  const buffer = await response.arrayBuffer();
  const bytes = new Uint8Array(buffer).slice(0, 4);
  const hex = Array.from(bytes).map(b => b.toString(16).padStart(2, '0')).join('');
  const validSignatures = {
    'ffd8ffe0': 'jpeg', 'ffd8ffe1': 'jpeg', 'ffd8ffe2': 'jpeg',
    '89504e47': 'png',
    '52494646': 'webp',
  };
  return Object.keys(validSignatures).some(sig => hex.startsWith(sig));
};
```

## Área 4: Thread Safety e Error Handling

### 4.1 `backend/app/services/face_detection_service.py` — Asyncio Lock

**Problema**: `_face_landmarker` é singleton global sem locking.

**Mudança**: Adicionar `asyncio.Lock()`:
```python
import asyncio

_face_landmarker_lock = asyncio.Lock()

class FaceDetectionService:
    async def detect_and_crop(self, base64_image: str) -> str:
        # ... decode image ...
        async with _face_landmarker_lock:
            result = _face_landmarker.detect(mp_image)
        # ... process landmarks ...
```

**NOTA**: MediaPipe `detect()` é CPU-bound. Em asyncio, isso bloqueia o event loop. Idealmente usar `asyncio.to_thread()` para rodar em thread separada:
```python
async with _face_landmarker_lock:
    result = await asyncio.to_thread(_face_landmarker.detect, mp_image)
```

Mas como `detect_and_crop` hoje é síncrono (`def`, não `async def`), primeiro precisamos convertê-lo para `async def` e atualizar os chamadores.

### 4.2 `backend/app/core/exceptions.py` — Criar (NOVO)

```python
from fastapi import HTTPException, status

class SanitizedHTTPException(HTTPException):
    """HTTP exception that never exposes internal details."""
    def __init__(self, status_code: int, public_message: str, internal_detail: str = ""):
        super().__init__(status_code=status_code, detail=public_message)
        self.internal_detail = internal_detail  # log only, never sent to client
```

### 4.3 `backend/app/services/analysis_service.py` — Usar SanitizedHTTPException

Substituir todos os `HTTPException` que expõem detalhes internos:
- `exc.response.text[:300]` → mensagem genérica
- `raw[:200]` → "Resposta inválida da API de análise"
