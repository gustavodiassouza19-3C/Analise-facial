# Plano: Tela de Perfil do Usuário — Integração AuthContext + ProfilePage

## Contexto

O AuthContext já busca o perfil do Supabase na autenticação (linha 15-19 do AuthContext.jsx), mas **não expõe o objeto `profile` completo** no contexto — apenas um `mergedUser` com campos limitados (`id, email, full_name, role, plan_type, subscription_status`). A ProfilePage, por sua vez, **re-busca o perfil do Supabase独立mente** (linha 38-43 do ProfilePage.jsx), causando:

1. **Request duplicada** — o perfil é buscado duas vezes (AuthContext + ProfilePage)
2. **Dados incompletos no contexto** — campos como `gender`, `age`, `style_objective`, `profile_picture` não estão disponíveis globalmente
3. **Sem necessidade de re-fetch manual** — ProfilePage poderia usar dados já carregados

## Alterações Propostas

### 1. `AuthContext.jsx` — Expor `profile` completo no contexto

**Arquivo:** `frontend/src/context/AuthContext.jsx`

- Adicionar `profile` ao `value` do Provider (linha ~133)
- Isso disponibiliza o objeto completo do perfil (`id, full_name, gender, age, style_objective, profile_picture, role, plan, status`) globalmente
- Sem quebras: o `mergedUser` continua existindo para compatibilidade

```diff
  value={{
    isAuthenticated: !!user,
    user: mergedUser,
+   profile,
    token,
    loading,
    ...
  }}
```

### 2. `ProfilePage.jsx` — Consumir perfil do contexto

**Arquivo:** `frontend/src/pages/ProfilePage.jsx`

- Usar `profile` do `useAuth()` em vez de buscar do Supabase no `useEffect`
- Inicializar os estados do formulário com os dados do contexto
- Adicionar função `refreshProfile()` para re-buscar do Supabase quando o usuário quiser atualizar
- Manter o estado `loading` para o caso de `profile` ainda estar `null` (carregamento inicial do AuthContext)
- Manter o `handleSave` com upsert no Supabase (já funciona corretamente)

**Fluxo:**
1. AuthContext carrega perfil → `profile` fica disponível
2. ProfilePage consome `profile` do contexto → popula formulário (sem request extra)
3. Usuário edita e clica "Salvar" → upsert no Supabase
4. Após salvar, opcionalmente atualizar o `profile` no contexto (chamar `fetchProfile` do AuthContext)

### 3. Atualização pós-save no AuthContext

**Arquivo:** `frontend/src/context/AuthContext.jsx`

- Expor `fetchProfile` no value do Provider para que outros componentes possam forçar uma atualização do perfil (ex: após salvar alterações na ProfilePage)

```diff
  value={{
    isAuthenticated: !!user,
    user: mergedUser,
    profile,
    token,
    loading,
    signUp,
    signIn,
    signOut,
    login: signIn,
    register: signUp,
+   refreshProfile: fetchProfile,
  }}
```
