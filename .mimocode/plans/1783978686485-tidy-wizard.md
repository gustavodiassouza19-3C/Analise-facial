# Plano: Corrigir erro ECONNREFUSED no backend

## Problema

O frontend está funcionando corretamente (proxy encaminha `/api/v1/auth/login` → `localhost:8000`), mas o **backend não existe** — nunca foi iniciado. O venv está vazio (só tem `pip`), nem PostgreSQL nem Redis estão instalados na máquina.

## Causas identificadas

1. **`backend/.venv` vazio** — `pip list` mostra apenas `pip` itself. Nenhuma dependência instalada.
2. **PostgreSQL não instalado** — `pg_isready` não encontrado, nenhum serviço PostgreSQL.
3. **Redis não instalado** — `redis-cli` não encontrado. (Mas Redis **não é usado** em nenhum código — só está no `config.py`).
4. **Backend nunca iniciado** — Nada escuta na porta 8000.

## Plano de correção

### Passo 1: Instalar dependências do backend no venv

```bash
cd backend
.venv\Scripts\activate
pip install -r requirements.txt
```

Isso instala FastAPI, SQLAlchemy, bcrypt, opencv, mediapipe, etc.

### Passo 2: Adicionar `aiosqlite` e migrar de PostgreSQL para SQLite

Como PostgreSQL não está instalado, o backend precisa usar SQLite para desenvolvimento.

**Arquivo: `backend/requirements.txt`** — adicionar:
```
aiosqlite>=0.20.0
```

**Arquivo: `backend/.env`** — trocar:
```
DATABASE_URL=sqlite+aiosqlite:///./facial_analysis.db
```

**Arquivo: `backend/app/core/config.py`** — trocar o default:
```python
DATABASE_URL: str = "sqlite+aiosqlite:///./facial_analysis.db"
```

**Arquivo: `backend/app/database/connection.py`** — adicionar `connect_args` para SQLite:
```python
engine = create_async_engine(
    settings.DATABASE_URL,
    echo=settings.DEBUG,
    pool_pre_ping=True,
    connect_args={"check_same_thread": False} if "sqlite" in settings.DATABASE_URL else {},
)
```

### Passo 3: Remover Redis do `.env` (opcional, apenas limpeza)

Redis não é usado em nenhum código — pode ignorar ou remover a linha do `.env`.

### Passo 4: Iniciar o backend

```bash
cd backend
.venv\Scripts\activate
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

### Passo 5: Testar

1. Criar conta em `http://localhost:5173/signup`
2. Login em `http://localhost:5173/login`
