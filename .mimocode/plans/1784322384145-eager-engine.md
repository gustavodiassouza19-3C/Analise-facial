# Plano: Rodar Frontend + Backend no Docker

## Contexto

A infraestrutura Docker já está pronta (docker-compose.yml, Dockerfiles, nginx.production.conf, .env.production.example). O problema é que o **Docker Desktop não está instalado** nesta máquina — o comando `docker` não é reconhecido.

## Passos para executar

### 1. Instalar Docker Desktop for Windows

Baixe em: https://docs.docker.com/desktop/install/windows-install/

- Marque "Use WSL 2 instead of Hyper-V" (recomendado)
- Reinicie o computador após a instalação
- Abra o Docker Desktop e aguarde ele ficar "Running" (ícone na bandeja)

### 2. Criar o arquivo .env.production

```powershell
cp .env.production.example .env.production
```

Preencher as variáveis obrigatórias no `.env.production`:

| Variável | Onde obter |
|----------|-----------|
| `POSTGRES_PASSWORD` | Criar uma senha forte (32+ chars) |
| `SECRET_KEY` | `python -c "import secrets; print(secrets.token_urlsafe(64))"` |
| `OPENROUTER_API_KEY` | https://openrouter.ai/keys |
| `VITE_SUPABASE_URL` | Dashboard Supabase > Settings > API |
| `VITE_SUPABASE_ANON_KEY` | Dashboard Supabase > Settings > API > anon public |

### 3. Build e start

```powershell
docker compose --env-file .env.production up -d --build
```

Isso vai:
- Baixar as imagens base (node, nginx, python, postgres, redis)
- Compilar o frontend (Vite → bundle JS)
- Compilar o backend (pip install + código)
- Criar os containers e conectá-los na rede `facemax`
- Criar volumes persistentes para PostgreSQL e Redis

### 4. Verificar

```powershell
docker compose ps
docker compose logs backend
```

- Frontend: http://localhost:3000
- Backend: http://localhost:8000
- API docs: http://localhost:8000/docs

### 5. Parar

```powershell
docker compose down
```

## Arquivos envolvidos (todos já criados)

- `docker-compose.yml` — orquestração dos 4 serviços
- `.env.production.example` — template das variáveis
- `frontend/Dockerfile` — build multi-stage (Node + Nginx)
- `frontend/nginx.production.conf` — proxy reverso + segurança
- `backend/Dockerfile` — build multi-stage (Python + Uvicorn)
- `backend/.dockerignore` — exclusões do contexto de build

## Nota

Não há código a ser escrito — toda a infraestrutura já existe. O único passo de instalação é o Docker Desktop.
